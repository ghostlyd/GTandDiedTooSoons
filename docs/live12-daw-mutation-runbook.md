# Live 12 DAW Mutation Runbook

Generated by `scripts/render_live12_daw_mutation_runbook.py` from the committed mutation contracts.

Status: `operator_runbook_not_applied`.

Do not commit Ableton sets, Max devices, source audio, renders, credentials, cookies, or license files.

## Queue Preparation

Run the full queue staging command when preparing the whole set:

```bash
python3 scripts/prepare_live12_daw_mutation_queue.py
```

All Ableton launches remain blocked until the launch command includes `--launch-ableton`, `--confirm-live-mutation`, `--operator-approval-reference`, and `--rollback-copy-reference`.

Public Domain Source Deck remains muted until provenance review.

## Approval Boundary

- Live-set mutation
- Max for Live device mutation
- private audio upload
- export or release

## Phase Contract

- `preflight`: Create the local mutation request and receipt template, then verify MIDI hashes before touching Live.
- `stage_import_bundle`: Copy the generated MIDI sketch into an ignored import bundle and inspect the launch plan.
- `apply_live_mutation`: Launch Ableton only after approval and rollback references are ready.
- `record_receipt`: Record what was applied, skipped, created, and redacted from operator evidence.
- `postflight`: Run repository validation and confirm no DAW binaries, source audio, renders, or credentials entered Git.
